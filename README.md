## 它做什么 · What It Does

大多数 AI 学习工具给你一个答案。MasterZhuyan 给你一套**研究过程**。

它的工作方式更像一位认真的导师：先把材料读透，把证据锁住，把知识结构建好——然后才开始写教学内容。输出不是一段对话，而是带引用溯源、章节结构和续学地图的**持久化文件包**。

> Most AI learning tools give you an answer. MasterZhuyan gives you a **research process**.
>
> It works like a rigorous tutor: read the sources first, lock the evidence, build the knowledge model — then write. The output isn't a chat reply; it's a **durable file package** with citation traces, structured chapters, and a continuation map.

## 核心流水线 · Core Pipeline

```
研究合约 + 动态研究树
Research Contract + Dynamic Research Tree
        │
        ▼
来源地图 + 来源卡片          Source Map + Source Cards
        │
        ▼
证据台账 + 证据卡片          Evidence Ledger + Evidence Cards
        │
        ▼
交叉核查 + 矛盾/缺口修复     Cross-check + Contradiction / Gap Repair
        │
        ▼
  ┌─────────────┐
  │ 锁定知识模型 │           LOCKED Knowledge Model
  └─────────────┘
        │
        ▼
章节计划                     Chapter Plan
        │
        ▼
每章 Teaching Composer       Per-chapter Teaching Composer Sidecars
        │
        ▼
Integrator 促进的证据绑定章节  Integrator-promoted Evidence-bound Chapters
        │
        ▼
质量溯源 + 最终合并           Quality Tracing + Final Merge
        │
        ▼
  final/final_merged.md  +  续学地图 Continuation Map
```

话题大小、来源多少、看起来简不简单——都不会绕开这条路，只会调整工件的规模。

> Topic size, source count, or apparent simplicity never bypass this route — they only tune artifact scale.

## 多智能体执行织物 · Multi-Agent Execution Fabric

每个 agent 有明确的边界任务，写有界工件，留可追溯的执行记录。Integrator 统一做促进/修订/丢弃决策。

> Each agent has a bounded scope, writes a bounded artifact, and leaves a traceable execution record. The Integrator owns all promote / revise / discard decisions.

| Agent | 职责 · Role |
|---|---|
| **Source Scout** | 主动采集来源，写来源卡片 · Active source acquisition, source cards |
| **Evidence Curator** | 从来源提取证据，写证据台账 · Evidence extraction from source cards |
| **Mechanism Modeler** | 建模机制、概念图、误解修复 · Mechanism modeling, concept graphs, misconception repair |
| **Teaching Composer** | 每章一个有界起草任务 · One bounded drafting task per chapter row |
| **Trace Curator** | 多 agent 状态汇总与续学映射 · Multi-agent state summary and continuation mapping |
| **Integrator** | 统一促进、溯源、合并 · Unified promotion, tracing, and final merge |

| 用户信号 · Signal | 章节重点 · Chapter Focus |

| `易错点` `陷阱` `容易混淆` `错题` | 错误模式、修复线索、触发提示 · Wrong patterns, repair cues, trigger clues |

## 命名备课模板 · Named Prep Templates

支持用户自定义命名的备课模板（如"某某式讲解"），作为软覆盖层影响来源优先级、讲解风格和章节侧重——不改变路由，不覆盖知识模型。

> User-defined named prep templates (e.g. "某某式讲解") act as soft overlays that shape source priority, explanation style, and chapter emphasis — without changing the route or overriding the Knowledge Model.

```bash
# 创建模板 · Create a template
python scripts/prep_template.py create "某某式讲解"

# 列出模板 · List templates
python scripts/prep_template.py list
```

## 质量护栏 · Quality Guardrails

执行中主动检查并记录以下问题 · Actively checked and recorded during execution:

| 问题 · Issue | 说明 · Description |
|---|---|
| **教学不足** Under-teaching | 事实被总结但没有解释连接关系 · Facts summarized without explaining connections |
| **假深度** Fake depth | 加了章节但没有解释价值 · Sections added without explanatory value |
| **虚假完整** False completeness | 关键参考框架或精度锚点被压缩掉 · Load-bearing frames or anchors omitted |
| **虚假精度** False precision | 没有接受证据的具体内容进入最终输出 · Specifics without accepted evidence in final output |
| **长文回避** Longform avoidance | 用对话替代了应有的持久化研究资产 · Chat substitutes for durable research asset |
| **模板囚禁** Template captivity | 命名模板让输出变得僵硬 · Named template makes output rigid |
