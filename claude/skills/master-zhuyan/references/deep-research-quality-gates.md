# MasterZhuyan 深度研究式学习质量门

本文件用于最终输出前的静默验证。它不是新的输出模板，也不要求把检查过程展示给用户；它的作用是防止“看起来很长、实际不稳”的学习材料进入最终交付。

适用场景：

1. 用户要求系统整理、深度讲解、研究式学习、长文、MD 文件、课程化输出。
2. 输入包含截图、上传文件、长材料、多来源材料、当前信息或高风险领域。
3. 主题需要机制推理、比较辨析、记忆支持、错因修复或专业边界。

## 使用原则

1. 先检查来源和证据，再检查教学表达。
2. 先保护事实边界，再追求讲得漂亮。
3. 每个结论都应能归入：来源事实、证据支持、教师综合、明确推断、未知/待核验。
4. 深度研究式学习不是堆章节；章节必须增加理解、记忆、判断或复用价值。
5. 高风险领域宁可标注不确定，也不能用无来源常识补齐关键建议。

## 1. 来源门

通过标准：

1. 已识别本轮可用来源类型：用户粘贴文本、图片/截图、上传文件、仓库文件、当前检索、稳定通识、用户指定教材或模板。
2. 已区分“本轮来源明确提供的内容”和“模型已有知识/教学综合”。
3. 对多文件或多材料，至少内部形成来源映射：来源 -> 主题 -> 可用证据 -> 缺口。
4. 对截图和图片，只使用可见信息；不可识别处标注：图片中该部分不清晰，无法准确识别。
5. 对当前信息、指南、法规、价格、版本、政策、标准、新闻等不稳定内容，已检索或明确说明需要当前来源验证。

失败处理：

1. 来源不足但可以教学时，只讲稳定概念结构，并标注“缺少本轮来源，具体细节不能确定”。
2. 来源矛盾时，先列出矛盾点，再进入教学综合；不要静默选择其中一个。
3. 来源缺失到无法完成用户目标时，询问用户补充材料或限定输出为概念框架。

## 2. 证据门

通过标准：

1. 核心定义、分类标准、机制链、阈值、公式、步骤、例外、风险、禁忌、适用条件都有来源或明确的稳定知识依据。
2. 关键数字、剂量、日期、指南年份、法律后果、价格、版本号、推荐等级、诊断/处置建议没有凭空生成。
3. 引用或转述材料时保留原有技术含义，不为了顺口而改写掉条件、例外或限定语。
4. 结论与证据粒度匹配：材料只支持“可能/相关/某条件下”，输出不能升级为“一定/必须/普遍”。
5. 反例、边界、例外被放在能影响判断的位置，而不是放在末尾象征性提醒。

失败处理：

1. 删除或降级无证据支撑的强结论。
2. 将缺口写成“原始材料未提供，不能确定”或“需要进一步核验”。
3. 对高风险细节，不用“常见情况”替代具体来源。

## 3. 推断标注门

通过标准：

1. 输出中能看出哪些是来源事实，哪些是教师解释、结构化重组、类比或推断。
2. 推断有触发依据，例如材料中的条件、定义、因果链、图表趋势或上下文。
3. 推断没有越过来源边界：从“材料提到 A 和 B 同时出现”不能直接推出“A 导致 B”。
4. 类比用于帮助理解，并明确类比边界；不能把类比当事实证据。
5. 不确定信息被显式标注，而不是用模糊语气掩盖。

失败处理：

1. 在相关段落补上“这是基于材料的推断”或“这里是教学类比，不是严格等同”。
2. 把无法支撑的推断改为问题、待核验点或学习提示。

## 4. 知识模型门

通过标准：

1. 主题有清晰的知识主干，而不是碎片堆叠。
2. 根据主题类型选择合适骨架：
   - 概念型：定义 -> 构成要素 -> 边界 -> 例子 -> 混淆点。
   - 机制型：基线 -> 扰动 -> 机制 -> 结果 -> 识别 -> 干预/边界。
   - 分类型：分类目的 -> 分支标准 -> 类别 -> 鉴别线索 -> 例外。
   - 流程型：目标 -> 标准 -> 步骤 -> 关键动作 -> 失败模式 -> 练习。
   - 公式型：条件 -> 变量 -> 推导直觉 -> 应用 -> 陷阱。
   - 规则/政策型：目的 -> 构成要件 -> 条件 -> 例外 -> 后果 -> 风险边界。
3. 章节之间有学习顺序：先建立定位，再解释机制，再进入比较、记忆、易错和应用。
4. 对复杂主题，能给出层级关系或判断链；对简单主题，不强行膨胀成空章节。

失败处理：

1. 若输出只是事实列表，补出最小因果链、分类依据或判断链。
2. 若章节重复，合并到能承载学习价值的章节。

## 5. 教学深度门

通过标准：

1. 用户问“为什么/机制/原理”时，回答包含起点、关键前提、机制变化、结果和边界。
2. 用户问“怎么区分/对比”时，回答包含决定性标准、相似原因、不同本质、误用后果和一句话区分。
3. 用户问“怎么用/怎么判断”时，回答包含可执行判断线索，而不只是定义复述。
4. 抽象概念至少有一个准确例子或边界案例；例子不应引入未经说明的新风险。
5. 对专业主题，解释“为什么这样判断”，不是只给结论。

失败处理：

1. 给机制缺口补一条“从条件到结果”的推演链。
2. 给比较缺口补一个“看到什么就归哪类”的判别线索。
3. 删除不增加理解的装饰性段落。

## 6. 记忆支持门

通过标准：

1. 对记忆、复习、考试、背诵、高频点请求，输出包含核心锚点、检索线索和 30 秒回顾。
2. 记忆钩子服务于准确性；不为了押韵牺牲概念边界。
3. 重要细节按“为什么值得记”组织，而不是平均铺开。
4. 对易混项，给出最短区分句或触发词。
5. 长文中有便于复习的压缩层：核心表、判断链、速记段或回顾清单。

失败处理：

1. 如果只有讲解没有回忆入口，补充记忆主干和 30 秒回顾。
2. 如果口诀可能误导，改用逻辑线。

## 7. 易错修复门

通过标准：

1. 至少指出最可能的误解、错因或混淆对象。
2. 每个关键易错点包含：错误说法/错误操作 -> 为什么容易错 -> 正确认识 -> 触发线索 -> 修正方法。
3. 对考试、临床、法律、金融、工程安全、网络安全等主题，明确不能外推的边界。
4. 对来源中的自相矛盾、图像不清、材料缺口，列为输出风险而不是隐藏。
5. 易错修复与主线相连，不做孤立的“注意事项”堆砌。

失败处理：

1. 补充至少一个会真正改变判断的易错点。
2. 把泛泛的“注意区别”改成具体触发线索。

## 8. 长文容器门

通过标准：

1. 默认文件化长文交付包含：manifest 或输出计划、index 或阅读顺序、章节文件、validation 记录、final/final_merged.md。
2. 章节命名能反映学习功能，例如定位、机制、比较、记忆、易错、应用、验证，而不是只有“第一章/第二章”。
3. 最终合并文件能独立阅读，不依赖聊天上下文才能理解。
4. validation 记录至少覆盖来源、证据、推断、缺口、长文完整性和风险边界。
5. 聊天最终回复只报告路径、阅读顺序、验证状态和已知限制；不粘贴整篇长文。

失败处理：

1. 缺少容器文件时补齐，而不是在聊天里补长篇正文。
2. 章节过多但学习价值重复时合并。
3. 用户明确要求 no files、brief、停止生成文件，或文件不可用时，说明文件化长文无法交付的原因，只给交付说明、限制说明或接续建议，不用聊天版替代完整学习材料。

## 9. 风险领域门

高风险领域包括但不限于：医学、法律、金融投资、工程安全、化学/生物安全、网络安全、心理健康、用药、急救、监管合规、儿童或弱势人群相关建议。

通过标准：

1. 输出定位为学习支持，不是个案诊断、处方、法律意见、投资建议或操作授权。
2. 当前指南、法规、标准、价格、版本、漏洞、政策等必须使用当前来源或标注需要核验。
3. 不给缺少条件的个案结论；必要条件缺失时列出“还需要哪些信息才能判断”。
4. 对可能造成伤害的步骤、剂量、阈值、处置方案、规避规则等，保持来源约束和安全边界。
5. 明确提醒专业场景需要合格专业人员或权威来源确认，但不把提醒当作替代解释。

失败处理：

1. 删除未经来源支持的具体建议。
2. 将结论改为学习框架、判断要素或待核验问题。
3. 对用户要求越界的部分，拒绝或转为安全的概念解释。

## 10. 最终通过判定

最终输出可以交付，必须同时满足：

1. 来源边界清楚。
2. 关键结论有证据或被标注为推断/不确定。
3. 知识主干足以支撑理解。
4. 用户显式学习意图得到强化：记忆、理解、比较、应用或易错修复。
5. 长文容器完整，或文件化交付受阻的限制说明清楚。
6. 高风险信息没有被无来源扩写成具体建议。
7. 最终回复简洁，只交代交付物、验证状态和限制。

任一硬门失败时，不要交付最终版；先修正文档或降低结论强度。
