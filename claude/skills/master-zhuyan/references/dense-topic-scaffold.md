# Dense topic scaffold

Use this file when the topic contains many mechanisms, branches, criteria, or exceptions. The goal is to prevent shallow summaries.

## Dense topic invariant

A dense explanation must preserve:

1. the central problem
2. the core entities
3. the causal or logical chain
4. the branch points
5. the criteria or thresholds
6. the exceptions
7. the failure modes or misconceptions
8. one concise transfer boundary

## Mechanism-heavy scaffold

Use this order:

1. What system is being discussed?
2. What normal baseline or default rule exists?
3. What disturbance or question changes the baseline?
4. Through which mechanism does the change propagate?
5. What observable result follows?
6. What test, calculation, sign, or criterion reveals it?
7. What intervention, method, or decision follows from the mechanism?
8. What similar case looks close but has a different mechanism?

## Branch-heavy scaffold

When there are many categories, build a branch map:

root concept -> branch criterion -> branch item -> key marker -> use/risk
(emit one row per source-supported branch; do not cap at three)

Each branch row carries its classification criterion, key marker, and use/risk so the criterion can flow into `required_anchors` or chapter prose.

## Formula-heavy scaffold

For formulas or models, explain:

1. what each variable means
2. what increases the result
3. what decreases the result
4. what assumptions must hold
5. what happens at boundaries or extremes
6. what common substitution or unit error occurs
7. how the formula guides action or interpretation

## Rule-heavy scaffold

For law, policy, protocols, or standards, explain:

1. the rule's purpose
2. who or what it applies to
3. elements that must be satisfied
4. exceptions
5. consequences
6. typical mistake
7. one practical judgment path

## Output rule

Use the scaffold to shape longform chapters. If the scaffold reveals too many branches, split them into useful chapters rather than compressing or replacing the deliverable with chat.
