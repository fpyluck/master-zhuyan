# Exemplar patterns

Use these as style anchors when longform chapters become over-rigid or generic. Do not copy the exact content unless it fits the user's task.

## 1. Understanding chapter pattern

User: 为什么负反馈能维持稳态？

Good longform shape:
Create a file-backed deliverable. The understanding chapter starts with the conclusion, then walks the chain:
目标范围存在 -> 感受器发现偏差 -> 调控中枢比较偏差 -> 效应器产生反向作用 -> 偏差缩小 -> 调节减弱

Include an easy-error section that repairs the misunderstanding that negative feedback "suppresses all activity"; it suppresses deviation from target range.

Why this works:
It keeps the one-route longform policy while preserving the strongest direct-explanation spine.

## 2. Comparison chapter pattern

User: AUC 和 accuracy 有什么区别？

Good longform shape:
Create a file-backed deliverable. The comparison chapter opens with the decisive distinction:
accuracy 评价某一个阈值下分类对了多少；AUC 评价所有可能阈值下正样本排在负样本前面的能力。

Then include use conditions, mechanism difference, common traps, and one-sentence distinction.

Why this works:
Comparison becomes a chapter emphasis, not a separate answer route.

## 3. Memory chapter pattern

User: 帮我记某分类知识

Good longform shape:
Create a file-backed deliverable. The memory chapter includes core positioning, memory skeleton, load-bearing details, retrieval hooks, mnemonic or logic line, easy errors, and 30-second recap.

Why this works:
The user gets durable recall without creating a separate chat route.

## 4. Delivery/limitation note pattern

User: 只在聊天里简短回答

Good answer shape:
State briefly that MasterZhuyan is a file-backed longform learning route and that this request stops the normal deliverable. Provide a short limitation or handoff note, not a full replacement lecture.

Why this works:
It prevents chat from becoming a second teaching route that bypasses source grounding, research planning, and final merge.
