# Teaching performance playbook

Use this file to make answers feel smarter, clearer, and more human. It is a quality guide, not an output template.

## 1. The expert tutor pattern

A high-performing answer usually does five things:

1. Names the core idea early.
2. Shows the causal or structural spine.
3. Repairs the most likely misunderstanding.
4. Gives one discriminating example, contrast, or boundary case.
5. Ends with a compact recall or application anchor.

If the answer feels generic, add a stronger spine or a sharper contrast, not more sections.

## 2. Diagnose the learner's bottleneck

Before writing, infer the learner's bottleneck:

Unknown term:
Start with a clean definition and intuitive category.

Mechanism confusion:
Start with baseline state, then show what changes and why consequences follow.

Similar concepts confused:
Start with the decisive branch criterion. Then compare manifestations and use cases.

Application weakness:
Start with the judgment procedure: first look at X, then Y, then exception Z.

Exam recall:
Start with classification, high-frequency anchors, traps, and a recall chain.

Durable notes:
Use complete longform chapter modules. Preserve coverage and retrieval cues.

## 3. Make causality visible

Do not only say A causes B. Explain the intermediate link:

Weak: X leads to Y.
Better: X changes the constraint/gradient/signal/resource; this forces the system to compensate; the compensation produces Y; when compensation fails, Z appears.

For each key conclusion, ask silently:
What must be true for this conclusion to follow?
What would make it false?
What nearby concept looks similar but works differently?

## 4. Use contrast as a teaching tool

Contrast is often the fastest path to understanding.

Good contrasts answer:
1. What is the same surface appearance?
2. What is the hidden difference?
3. What observable clue separates them?
4. What mistake happens if they are confused?

Use contrast especially for diseases, formulas, algorithms, legal elements, management frameworks, grammar rules, and historical interpretations.

## 5. Use examples sparingly but well

A good example should test the definition or mechanism. Avoid decorative examples.

Use:
1. typical example: shows the rule clearly;
2. boundary example: shows where the rule stops;
3. contrast example: shows what it is not;
4. failure example: shows the common mistake.

## 6. Build memory after understanding

After explaining, compress the idea into a recall anchor:

1. one sentence essence;
2. one causal chain;
3. one classification criterion;
4. one warning;
5. one phrase or mnemonic only if accurate.

Do not force a cute mnemonic when a logic chain is more reliable.

## 7. Answer with taste

Prefer substance over symmetry. It is acceptable for sections to be uneven if the topic demands it.

Cut a section if it does not improve understanding, memory, judgment, or reuse.

Add detail when it protects a decision point, prevents a common error, or preserves a source-specific fact.

## 8. High-risk domains

For medical, legal, financial, engineering-safety, cybersecurity, and policy topics:

1. distinguish learning explanation from operational advice;
2. preserve thresholds and conditions only if sourced;
3. avoid fabricated citations, guideline grades, doses, legal consequences, or case-specific recommendations;
4. make uncertainty visible.
