# Exemplar patterns

Use these as style anchors when longform chapters become over-rigid or generic. Borrow the move, not the example content. File-backed delivery is set by `SKILL.md`; these patterns only show chapter emphasis inside that route.

## 1. Understanding chapter pattern

User: 为什么负反馈能维持稳态？

Good chapter move:
Start with the conclusion, then walk the chain:
目标范围存在 -> 感受器发现偏差 -> 调控中枢比较偏差 -> 效应器产生反向作用 -> 偏差缩小 -> 调节减弱

Route the misunderstanding that negative feedback "suppresses all activity" into the chapter's easy-error repair; it suppresses deviation from target range.

## 2. Comparison chapter pattern

User: AUC 和 accuracy 有什么区别？

Good chapter move:
Open with the decisive distinction:
accuracy 评价某一个阈值下分类对了多少；AUC 评价所有可能阈值下正样本排在负样本前面的能力。

Then add use conditions, mechanism difference, common traps, and one-sentence distinction.

## 3. Memory chapter pattern

User: 帮我记某分类知识

Good chapter move:
Build the memory chapter from core positioning, memory skeleton, load-bearing details, retrieval hooks, mnemonic or logic line, easy errors, and 30-second recap.

## 4. File output unavailable pattern

User: 只在当前对话里回答，不生成文件

Good answer shape:
Return a delivery or limitation note only: state that the file-backed DeepResearch asset was not created, name the missing or blocked capability, preserve any already-created paths if they exist, and give the smallest useful continuation step.

Why this works:
It keeps chat as coordination around the durable deliverable, not as a substitute teaching surface.
